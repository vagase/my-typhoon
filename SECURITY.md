# 安全政策

## 支持范围

项目当前仅维护 `main` 分支的最新版本。旧提交和第三方自行分发的修改版本不提供安全更新保证。

## 报告漏洞

请优先使用 GitHub 仓库的 **Security → Report a vulnerability** 私下报告漏洞。不要在公开 Issue、讨论或截图中披露：

- 精确位置或可识别个人的信息；
- API 密钥、签名证书或访问令牌；
- 尚未修复的可利用细节；
- 第三方服务返回的敏感日志。

报告建议包含受影响版本、复现步骤、影响评估和可能的修复方向。维护者将在合理时间内确认报告并协调披露；本项目不承诺赏金或固定响应时限。

## 第三方服务问题

如果问题只存在于中央气象台、Open-Meteo 或 Apple 服务本身，请同时遵循相应服务的安全报告流程。不要通过本项目尝试绕过第三方访问控制、限流或服务条款。
