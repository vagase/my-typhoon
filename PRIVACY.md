# 隐私说明

最后更新：2026 年 7 月 19 日

TyphoonBar 不要求账号，不集成广告、用户画像或第三方分析 SDK，也不由项目维护者运营数据收集服务器。

## 使用的数据

### 当前位置

只有在用户通过 macOS 系统授权后，应用才会读取当前位置。位置用于：

- 计算用户与台风路径的距离和主要影响日期；
- 获取当地逐小时风雨模式预报；
- 将坐标转换成便于理解的地区名称。

位置坐标保存在应用进程内存中，应用退出后不再保留。本项目代码不会把位置写入文件、UserDefaults、iCloud 或项目维护者控制的服务器。

### 向第三方发送的数据

为了提供功能，应用会产生以下第三方网络请求：

- 将经纬度发送给 Open-Meteo，以获取当地天气和台风周边模式风场；
- 通过 Apple 的 Core Location/CLGeocoder 服务解析当前位置名称；
- 通过 Apple MapKit 加载地图内容；
- 向中央气象台读取公开台风数据。

这些服务可能根据各自政策处理 IP 地址、请求时间、坐标和其他必要的网络元数据：

- [Open-Meteo Privacy Policy](https://open-meteo.com/en/terms#privacy)
- [Apple Privacy Policy](https://www.apple.com/legal/privacy/)
- [中央气象台网站声明](https://www.nmc.cn/publish/cms/view/722665831f0a4e98800c41e691444963.html)

## 保存与删除

TyphoonBar 使用临时网络会话，不建立项目账号或云端用户档案。应用内存中的位置和天气响应会在进程结束后释放。由于项目维护者不接收或保存用户位置，因此没有由项目维护者执行的服务器端删除流程。

第三方服务的日志保留由各服务自己的政策决定。

## 控制权限

用户可以拒绝定位权限；台风目录和路径仍可查看，但依赖位置的日期、距离和当地影响不可用。用户可以在 macOS“系统设置 → 隐私与安全性 → 定位服务”中随时撤回权限。

## 儿童隐私

应用不面向儿童收集个人信息，不包含广告或面向儿童的账号功能。

## 问题反馈

隐私或安全问题请通过 GitHub Security Advisory 私下联系项目维护者，不要在公开 Issue 中提交精确位置、网络地址或其他敏感数据。
